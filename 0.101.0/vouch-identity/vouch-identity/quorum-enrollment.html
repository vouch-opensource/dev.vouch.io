Quorum Enrollment

1. Initialize the Vouch Identity Client​

Set up your Vouch Identity client in your admin/backend environment:

// Example: JavaScript/TypeScript
import { VouchClient } from 'vouch-identity-sdk';

const vouchClient = new VouchClient({
  apiKey: 'YOUR_API_KEY',
  // ...other configuration
});

2. Create a Quorum Enrollment Session​

When starting a new quorum enrollment (e.g., for a candidate user), call the SDK to initialize the process. This returns a unique enrollmentSessionId and possibly a shareable link or code:

const { enrollmentSessionId, enrollmentShareUrl } =
  await vouchClient.createQuorumEnrollmentSession({
    name: "Jane Doe",
    email: "jane.doe@example.com",
    // Additional metadata if needed
  });
  • enrollmentSessionId uniquely identifies this enrollment.
  • enrollmentShareUrl is a link that can be sent to approvers for voting.

3. Distribute the Enrollment for Voting​

Share the enrollmentShareUrl or enrollmentSessionId with the required voters. Depending on your implementation, they may:

  • Receive an email with a unique link.
  • Visit a protected admin page in your app.
  • Use a separate interface to cast their vote.

4. Approvers Cast Their Votes​

Each approver uses the SDK (or a custom interface you’ve built around it) to vote. Here’s an example of how an approver might cast an approve vote:

await vouchClient.voteQuorumEnrollmentSession({
  enrollmentSessionId,
  vote: 'approve',         // "approve" or "reject"
  voterId: 'teamLead123',  // Unique identifier for the approver
  reason: 'All docs verified in person'
});

If an approver needs to reject for any reason:

await vouchClient.voteQuorumEnrollmentSession({
  enrollmentSessionId,
  vote: 'reject',
  voterId: 'teamLead456',
  reason: 'Insufficient documentation'
});

5. Check Quorum Threshold & Finalize Enrollment​

Once enough approvers have approved (meeting the quorum threshold), finalize enrollment:

const result = await vouchClient.finalizeQuorumEnrollment(enrollmentSessionId);

if (result.status === 'ENROLLED') {
  console.log('Enrollment finalized successfully!');
} else {
  console.log('Enrollment still pending or rejected.');
}
  • If the threshold is met, Vouch Identity finalizes the user’s account and grants them access.
  • If votes are still pending or a reject threshold is triggered, the enrollment remains in a pending or failed state.

6. (Optional) Tie a Device to the Enrollment​

If your workflow requires the newly enrolled user to register their device immediately, you can integrate a QR-based or code-based process (similar to In Person Enrollment):

const { deviceRegistrationUrl } =
  await vouchClient.initiateDeviceRegistration({
    userId: 'USER_ID_RETURNED_FROM_FINALIZE',
  });

// Display or share the deviceRegistrationUrl for the user to scan and register

Summary​

  1. Create Quorum Session: Set up a new quorum enrollment session, capturing the user’s details.
  2. Distribute: Send the enrollment ID or share link to the required approvers.
  3. Vote: Approvers call the SDK to approve or reject.
  4. Finalize: Once the quorum threshold is met, finalize enrollment via finalizeQuorumEnrollment().
  5. Manage & Assign: Optionally register devices, assign roles, and manage permissions using other SDK methods.

This approach ensures that multiple stakeholders can weigh in before a new user is fully enrolled, adding a layer of oversight and security to your onboarding process.
