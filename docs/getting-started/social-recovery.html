Social Recovery

The Vouch Identity Platform has a new approach to account recovery. Rather than going through a centralized IT or HR support channel, team members vouch for other team members if a device is lost or stolen. If a role is more secure or sensitive, rules can be made to require multiple team members to vote the person back onto the team.

Step 1: Initiate Recovery​

The process begins when the user who lost their device (e.g., Jeff) downloads and installs the Vouch Identity app on a new device.

import { VouchSDK } from 'vouch-identity-sdk';

const vouch = new VouchSDK();

// Step 1: Initialize the recovery process
async function initiateRecovery(userId) {
    const recoverySession = await vouch.startRecovery(userId);
    console.log(`Recovery session started: ${recoverySession.id}`);
    return recoverySession;
}

Step 2: Generate a QR Code for Secure Channel Bootstrap​

The new device generates a QR code that encodes bootstrap data for secure communication with approvers.

async function generateQRCode(recoverySession) {
    const qrCodeData = await vouch.generateRecoveryQRCode(recoverySession.id);
    // Display this QR code in the app's UI
    console.log('QR Code Data:', qrCodeData);
}

Step 3: Approvers Validate and Endorse​

Team members (approvers) scan the QR code using their Vouch Identity app to validate the identity and approve the recovery.

async function approveRecovery(approverId, recoverySession) {
    const endorsement = await vouch.approveRecovery({
        approverId,
        sessionId: recoverySession.id,
    });
    console.log(`Recovery approved by ${approverId}`);
    return endorsement;
}

Approvals can be configured to require a quorum of approvers. This can be done via the SDK's configuration:

vouch.setApprovalPolicy({
    quorum: 2, // Minimum number of approvers required
});

Step 4: Confirm Recovery and Assign the New Device​

Once the quorum is reached, the SDK finalizes the recovery process and assigns the new device to the user.

async function finalizeRecovery(recoverySession, newDeviceId) {
    const result = await vouch.finalizeRecovery({
        sessionId: recoverySession.id,
        newDeviceId,
    });
    console.log(`Device recovery finalized: ${result.status}`);
}

Step 5: Disable Lost Device (Optional)​

The organization may optionally disable the old device once the recovery process is complete.

async function disableLostDevice(oldDeviceId) {
    const result = await vouch.disableDevice(oldDeviceId);
    console.log(`Old device disabled: ${result.status}`);
}

Implementation Notes​

  1. Security: All communication between devices and approvers is encrypted end-to-end.
  2. Physical Presence: Physical proximity between the lost device owner and approvers is recommended for secure endorsements.
  3. Quorum Flexibility: For highly sensitive environments, the quorum size can be adjusted to increase security.

Sample Workflow​

  1. User (Jeff) starts the recovery process by generating a QR code on their new device.
  2. Approvers (e.g., Jane and Jennifer) scan the QR code, validate Jeff's identity, and endorse the recovery.
  3. Once the quorum is met, the recovery session is finalized, and the new device is activated.
  4. Optionally, the old device is disabled to prevent unauthorized access.

This SDK-driven process ensures that device recovery is both secure and user-friendly, empowering organizations to maintain trust while safeguarding sensitive resources.
